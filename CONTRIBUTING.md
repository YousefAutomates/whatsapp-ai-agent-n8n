# Contributing Guidelines

Thank you for your interest in contributing to this project!

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs
- Describe the problem clearly with steps to reproduce
- Include your n8n version and any error messages

### Suggesting Improvements
- Open an Issue with the `enhancement` label
- Describe your idea and its benefits

### Submitting Changes
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Test thoroughly
5. Submit a Pull Request

## Workflow Contribution Guidelines
- Always remove credentials before sharing workflows
- Add sticky notes to explain complex logic
- Test with both text and non-text messages
- Ensure Arabic text renders correctly
